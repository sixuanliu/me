CSS

Silverwhite uses Sass instead of Less with the following changes:

  • Alphabetical property ordering.
  • Converted variable and mixin names from camelCase to hyphens.
  • Removed $font-size-large, $font-size-small, $font-size-mini, $btn-border, $icon-sprite-path, $icon-white-sprite-path, $pagination-border, $warning-border, $error-border, $success-border, $info-border variables.
  • Removed $btn-*-background-highlight variables and used $btn-*-background-hover instead.
  • Removed CSS3 mixins and used Compass's instead
  • Removed reset-filter and gradient-bar mixins which has merged into the button-background mixin.
  • Added font-size-* mixins.

Bootstrap forces the heading level of many components such as .alert h4. Silverwhite does not have this limit by applying the heading styles to h1-h6 where appropriate, or by adding an additional class, .hero-unit-heading.

Navbar dropdown on hover

Silverwhite has pure CSS dropdown on hover for Navbar. To use it, simply add .dropdown-hover to .dropdown.

Title
  • Dropdown
    • Action
    • Another action
    • Something else here
    • Separated link
<ul class="nav">
  <li class="dropdown dropdown-hover">
    <a href="#" class="dropdown-toggle">Dropdown <b class="caret"></b></a>
    <ul class="dropdown-menu">
      ...
    </ul>
  </li>
</ul>

Add the following JavaScript code to prevent the page jumps to top when clicking on href="#" links.

$('body').on('click', '[href="#"]', function(event) {
  event.preventDefault();
});

You may want to change 'body' to a more specify container (eg. '.dropdown-hover') if jump to top via <a href="#"> is used somewhere on the page.

For responsive site the menu should be placed inside .nav-collapse otherwise it breaks on small screens.

Not all components work in mega menu such as .nav-{tabs,pills,stacked}, Portfolio filter, FlexSlider, Google Maps and Carousel.

Bootstrap's dropdown plugin (data-toggle="dropdown") closes the menu on clicking inside. You may use .dropdown-hover or add the following JavaScript code (reference) to keep it there.

$('.dropdown-menu[class*="span"]').click(function(event) {
  event.stopPropagation();
});

Font Awesome

Silverwhite replaced the Glyphicons with Font Awesome. No additional CSS file needed except for IE7.

<!--[if IE 7]>
  <link rel="stylesheet" href="assets/css/font-awesome-ie7.min.css">
<![endif]-->

Web fonts are rendering bad in Chrome under Windows. Silverwhite added the following code that use SVG font for smooth rendering and the media query avoids layout breaking. (reference)

@media screen and (-webkit-min-device-pixel-ratio: 0) {
  @font-face {
    font-family: 'FontAwesome';
    src: url("../font/fontawesome-webfont.svg#fontawesomeregular") format("svg");
  }
}

The font files are placed in assets/font/.

Colors

The files in assets/css/ are of the blue color. The CSS files of red and green are located in assets/css-red/ and assets/css-green/ respectively. The corresponding Sass directories are assets/sass-red/ and assets/sass-green/ which only override variables. It's not a usual way to override Sass variables but for having separated CSS directories for the additional colors.

HTML template

Silverwhite uses this template for the documentation and page templates:

<!DOCTYPE html>
<html class="no-js" lang="en">
<head>
  <meta charset="utf-8">

  <title>Silverwhite</title>

  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- http://www.456bereastreet.com/archive/201209/tell_css_that_javascript_is_available_asap/ -->
  <script>
    document.documentElement.className = document.documentElement.className.replace(/(\s|^)no-js(\s|$)/, '$1js$2');
  </script>

  <!-- Exclude IE7 becuase it may renders web fonts blurry -->
  <!--[if !(IE 7)]><!-->
    <link rel='stylesheet' href='http://fonts.googleapis.com/css?family=Open+Sans:400italic,700italic,400,700'>
  <!--<![endif]-->

  <link rel="stylesheet" href="assets/css/bootstrap.css">
  <link rel="stylesheet" href="assets/css/bootstrap-responsive.css">
  <!--[if IE 7]>
    <link rel="stylesheet" href="assets/css/font-awesome-ie7.min.css">
  <![endif]-->

  <!--[if lt IE 9]>
    <script src="assets/js/html5shiv.min.js"></script>
  <![endif]-->
</head>
<body>

  ...

  <script src="assets/js/jquery.min.js"></script>
  <script src="assets/js/bootstrap.min.js"></script>
</body>
</html>

The only use of the no-js trick in Silverwhite is for FlexSlider to show the first slide if JavaScript is disabled.

<link rel='stylesheet' href='http://fonts.googleapis.com/css?family=Open+Sans:400italic,700italic,400,700'>

This line embed the Open Sans web font which Silverwhite uses (body { font-family: "Open Sans", Arial, sans-serif; }). You may want to custom the styles and character sets included.
